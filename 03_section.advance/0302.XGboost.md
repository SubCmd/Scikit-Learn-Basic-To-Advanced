## 1. 개념 및 정의
 
### 1-1. 부스팅이란?
 
**이전 모델이 틀린 데이터에 가중치를 부여하며 순차적으로 학습하는 앙상블**
배깅(병렬)과 달리 **순차적(Sequential)**으로 모델을 쌓음

```
배깅 (랜덤 포레스트):        부스팅 (XGBoost):
트리1 ─┐                    트리1 → 잔차 계산
트리2 ─┤→ 투표 → 예측        트리2(잔차 학습) → 잔차 계산
트리3 ─┤                    트리3(잔차 학습) → ...
트리4 ─┘                    → 모든 트리의 예측을 합산 → 최종 예측
(병렬, 독립)               (순차, 보완)
```
 
| 비유 | 배깅 | 부스팅 |
|------|------|--------|
| 학습 방식 | 학생 100명이 각각 독립적으로 풀고 투표 | 한 학생이 틀린 문제를 다음 학생이 집중 학습 |
| 효과 | 분산(variance) 감소 | **편향(bias) 감소** |

### 1-2. XGBoost vs. LightGBM vs. CatBoost

| 항목 | XGBoost | LightGBM | CatBoost |
|------|---------|----------|----------|
| 트리 성장 | Level-wise (균형) | **Leaf-wise** (비균형) | Level-wise (변형) |
| 속도 | 보통 | **가장 빠름** | 보통 |
| 메모리 | 보통 | **적음** | 많음 |
| 범주형 처리 | 수동 인코딩 필요 | **자체 지원** | **자체 지원** |
| Kaggle 인기 | ★★★★★ | ★★★★★ | ★★★★ |
| 설치 | `pip install xgboost` | `pip install lightgbm` | `pip install catboost` |

> **실무 권장**
> 정형 데이터(tabular)에서는 **LightGBM을 기본값**으로 시작하고, XGBoost와 비교하는 것이 효율적!

#### Level-wise vs Leaf-wise
 
```
Level-wise (XGBoost):       Leaf-wise (LightGBM):
     [root]                      [root]
    /      \                    /      \
  [L1]    [L1]               [L1]    [L1]
  / \     / \                 / \
[L2][L2][L2][L2]           [L2][L2]    ← 손실이 큰 리프만 분할
                                        → 더 적은 분할로 더 좋은 성능
```

### 1.3 핵심 하이퍼파라미터
 
#### XGBoost / LightGBM 공통 (이름은 다르지만 개념 동일)
 
| 개념 | XGBoost | LightGBM | 설명 | 튜닝 가이드 |
|------|---------|----------|------|-------------|
| 트리 수 | `n_estimators` | `n_estimators` | 부스팅 라운드 수 | 100~1000 |
| 학습률 | `learning_rate` | `learning_rate` | 각 트리의 기여도 | **0.01~0.3** |
| 최대 깊이 | `max_depth` | `max_depth` | 트리 깊이 제한 | 3~10 (XGB), -1(LGB) |
| 서브샘플 | `subsample` | `subsample` | 각 트리의 샘플 비율 | 0.6~1.0 |
| 특성 서브샘플 | `colsample_bytree` | `colsample_bytree` | 특성 비율 | 0.6~1.0 |
| L1 규제 | `reg_alpha` | `reg_alpha` | Lasso 규제 | 0~10 |
| L2 규제 | `reg_lambda` | `reg_lambda` | Ridge 규제 | 0~10 |
| 리프 수 | — | `num_leaves` | LightGBM 핵심 | **31** (기본), 20~150 |


#### learning_rate와 n_estimators의 관계

```
learning_rate 작음(0.01) + n_estimators 많음(1000) → 천천히 정교하게 학습 (느리지만 안정적)
learning_rate 큼(0.3)   + n_estimators 적음(100)  → 빠르지만 거칠게 학습 (빠르지만 과적합 위험)
```
 
> **실무 팁**
> `learning_rate=0.1`, `n_estimators=300` 정도로 시작하고, Early Stopping으로 최적 n_estimators를 자동 탐색한다.
