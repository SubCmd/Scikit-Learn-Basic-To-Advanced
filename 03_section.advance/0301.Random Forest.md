# Phase 3-1: Random Forest (랜덤 포레스트)

## 1. 개념 및 정의

### 1-1. 앙상블 학습이란?
여러 개의 약한 모델(weak learner)을 결합하여 하나의 강한 모델(strong learner)을 만드는 방법

| 비유 | 설명 | 
| ---- | ---- | 
| 집단 지성 | 한 명의 전문가보다 100명의 의견을 종합하는 것이 더 정확 | 
| 포트폴리오 | 하나의 주식에 올인하지 않고, 여러 종목에 분산 투자하는 것 | 

#### 앙상블의 2대 전략
```
앙상블
├── 배깅 (Bagging) ← 랜덤 포레스트
│   └── 여러 모델을 병렬로 학습, 결과를 투표/평균
│   └── 분산(variance)을 줄임 → 과적합 방지
│
└── 부스팅 (Boosting) ← XGBoost, LightGBM (3-2에서 학습)
    └── 이전 모델의 실수를 보완하며 순차 학습
    └── 편향(bias)을 줄임 → 정확도 향상
```

---

### 1-2. 랜덤 포레스트?
여러 개의 결정 트리를 무작위로 생성하고, 다수결(분류) 또는 평균(회귀)으로 예측하는 배깅 앙상블.

```
랜덤 포레스트 = 배깅(Bootstrap Aggregating) + 무작위 특성 선택
 
데이터셋 → [부트스트랩 샘플 1] → 트리 1 → 예측 1 ─┐
        → [부트스트랩 샘플 2] → 트리 2 → 예측 2 ─┤→ 다수결/평균 → 최종 예측
        → [부트스트랩 샘플 3] → 트리 3 → 예측 3 ─┤
        → ...                                     ─┘
```

#### 무작위성 2가지

| 무작위성 | 방법 | 효과 | 설명 | 
|----------|------|------|-----|
| **부트스트랩 샘플링** | 원본에서 중복 허용 랜덤 추출 | 각 트리가 다른 데이터로 학습 | 주어진 하나 데이터셋에서 복원 추출 사용해 여러 개 새로운 표본을 만들어냄 |
| **무작위 특성 선택** | 각 분할에서 일부 특성만 고려 | 트리 간 상관관계 감소 → 다양성 | |
 
> **핵심**
> 개별 트리는 부정확할 수 있지만, 많은 트리의 "집단 지성"이 정확도를 높인다.

### 1-3. 결정 트리 vs. 랜덤 포레스트

| 항목 | 결정 트리 | 랜덤 포레스트 |
|------|-----------|--------------|
| 모델 수 | 1개 | N개 (기본 100개) |
| 과적합 | **취약** | 강건 (배깅 효과) |
| 분산 (Variance) | 높음 | **낮음** |
| 해석 가능성 | 높음 (규칙 추출) | 낮음 (블랙박스) |
| 성능 | 보통 | **높음** |
| 스케일링 | 불필요 | 불필요 |
| 학습 속도 | 빠름 | 느림 (병렬 처리 가능) |
| 특성 중요도 | 불안정 | **안정적** |

---

### 1.4 핵심 하이퍼파라미터
 
| 파라미터 | 설명 | 기본값 | 튜닝 가이드 |
|----------|------|--------|-------------|
| `n_estimators` | 트리 수 | 100 | 많을수록 안정적 (100~500) |
| `max_depth` | 트리 최대 깊이 | None | None(제한 없음) 또는 10~30 |
| `min_samples_split` | 분할 최소 샘플 | 2 | 과적합 시 5~20으로 증가 |
| `min_samples_leaf` | 리프 최소 샘플 | 1 | 과적합 시 2~10으로 증가 |
| `max_features` | 분할 시 고려 특성 수 | 'sqrt' | 분류: 'sqrt', 회귀: 1.0(전체) |
| `n_jobs` | 병렬 CPU 수 | None | **-1** (전체 CPU 사용) |
| `oob_score` | OOB 평가 사용 | False | True 권장 (무료 검증 세트) |

#### OOB (Out-of-Bag) Score
 
부트스트랩 샘플링에서 **선택되지 않은 약 37%의 데이터**를 자동 검증 세트로 활용한다.
별도의 검증 세트 없이 모델 성능을 추정할 수 있다.
