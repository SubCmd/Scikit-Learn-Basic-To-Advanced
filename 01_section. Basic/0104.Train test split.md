# Phase 1-4: Train Test Split

## 1. 개념 및 정의

### 1-1. 왜 데이터를 분리하는가?
모델의 **일반화 성능(Generalization Performance)**을 평가하기 위해서

| 상황 | 비유 | 문제점 |
|------|------|--------|
| 훈련 데이터로만 평가 | 시험 전에 기출문제로 공부하고, 같은 기출문제로 시험 보기 | 실력이 아닌 암기력을 측정 → **과적합** |
| 훈련/테스트 분리 후 평가 | 기출문제로 공부하고, **새로운 문제**로 시험 보기 | 진짜 실력(일반화 능력)을 측정 |

> PM 관점: 과거 3개월 데이터로 이탈 예측 모델을 만들었다면, 최근 1개월 데이터로 모델이 실제로 잘 예측하는지 검증해야 함.
> 과거 데이터에서만 잘 맞추고 미래 데이터에서 못 맞추면 실무에서 쓸모 없음.

---

### 1-2. 데이터 분리 전략

```
전체 데이터 (100%)
├── 훈련 세트 (Training Set): 70~80%  → 모델 학습
└── 테스트 세트 (Test Set): 20~30%    → 최종 성능 평가

더 엄밀한 분리:
전체 데이터 (100%)
├── 훈련 세트 (Training Set): 60~70%  → 모델 학습
├── 검증 세트 (Validation Set): 10~15% → 하이퍼파라미터 튜닝
└── 테스트 세트 (Test Set): 15~20%    → 최종 성능 평가
```

| 데이터셋 | 역할 | 비유 |
|----------|------|------|
| **훈련 세트** | 모델이 패턴을 학습 | 교과서로 공부 |
| **검증 세트** | 학습 중 하이퍼파라미터 튜닝 | 모의고사로 실력 점검 |
| **테스트 세트** | 최종 성능 평가 (한 번만!) | 수능 본시험 |

> 주의: 테스트 세트는 최종 평가 1회에만 사용해야 한다.
> 테스트 결과를 보고 모델을 수정하면, 테스트 세트에 대한 과적합이 발생함.

---

### 1-3. train_test_split 핵심 파라미터

| 파라미터 | 설명 | 기본값 | 권장값 |
|----------|------|--------|--------|
| `test_size` | 테스트 세트 비율 | 0.25 | **0.2** (80:20 분리) |
| `random_state` | 랜덤 시드 (재현성) | None | **42** (또는 아무 정수) |
| `shuffle` | 분리 전 셔플 여부 | True | True |
| `stratify` | 클래스 비율 유지 | None | **분류 문제에서 y 지정** |

---

### 1-4. stratify란? (분류 문제 필수)
정의: 원본 데이터의 클래스 비율을 훈련/테스트 세트에 동일하게 유지.

```
원본 데이터: A클래스 70%, B클래스 30%
 
stratify 미사용 (무작위 분리):
  훈련: A 75%, B 25%  ← 비율이 달라질 수 있음
  테스트: A 50%, B 50% ← 비율이 달라질 수 있음
 
stratify=y 사용:
  훈련: A 70%, B 30%  ← 원본 비율 유지!
  테스트: A 70%, B 30% ← 원본 비율 유지!
```

| stratify 사용 | 미사용 |
|---------------|--------|
| 훈련/테스트의 클래스 비율 일치 | 무작위 편향 가능 |
| 특히 **불균형 데이터**에서 중요 | 소수 클래스가 테스트에 없을 수도 있음 |
| 분류 문제에서 **항상 사용 권장** | 회귀 문제에서는 불필요 |

-------------------------------------------------------------------------------------------------------

### 1-5. random_state란? (재현성)

`random_state`는 랜덤 분리의 시드(seed) 값이다. 같은 값을 넣으면 항상 같은 결과가 나옴.

```
random_state=42로 실행 → 항상 동일한 분리 결과
random_state=None      → 실행할 때마다 다른 결과
 
→ 실험의 재현성(reproducibility)을 위해 반드시 설정할 것
→ 숫자 자체는 의미 없음 (42가 관례적으로 많이 사용됨)
```

### 1-6. 분리 비율 가이드

| 전체 데이터 크기 | 권장 비율 | 이유 |
|-----------------|----------|------|
| 소규모 (< 1,000) | 80:20 또는 70:30 | 학습 데이터 부족 방지 |
| 중규모 (1,000 ~ 100,000) | 80:20 | 일반적 기준 |
| 대규모 (> 100,000) | 90:10 또는 95:5 | 테스트 세트도 충분히 큼 |
