# Phase 1-3: 데이터 로딩 & 탐색 (EDA 기초)

## 1. 개념 및 정의

### 1-1. EDA란?
EDA(Exploratory Data Analysis, 탐색적 데이터 분석)는 모델링 전에 데이터의 구조, 분포, 이상치, 결측치, 변수 간 관계를 파악하는 과정

#### 왜 EDA가 중요한가?
| 원칙 | 설명 |
|------|------|
| **Garbage In, Garbage Out** | 나쁜 데이터를 넣으면 나쁜 모델이 나온다 |
| **80/20 법칙** | ML 프로젝트의 80%는 데이터 이해/전처리에 소요된다 |
| **도메인 이해** | 데이터를 보지 않고 모델을 만들면, 비즈니스 맥락을 놓친다 |

#### ML 프로젝트에서 EDA의 위치
```
비즈니스 문제 정의 → 데이터 수집 → [EDA] → 전처리 → 모델링 → 평가 → 배포
                                   ↑
                              현재 학습 단계
```

### 1-2. EDA 체크리스트
ML을 위한 EDA는 다음 5가지를 반드시 확인한다:

| # | 체크 항목 | 확인 방법 | 왜 중요한가 |
|---|----------|-----------|-------------|
| 1 | **Shape** (크기) | `df.shape` | 샘플 수 vs 특성 수 비율 파악 |
| 2 | **데이터 타입** | `df.dtypes`, `df.info()` | 수치형/범주형 구분 → 전처리 전략 결정 |
| 3 | **결측치** | `df.isnull().sum()` | 결측치 처리 전략 결정 (삭제/대체) |
| 4 | **기초 통계량** | `df.describe()` | 이상치, 스케일 차이, 분포 확인 |
| 5 | **타겟 분포** | `df['target'].value_counts()` | 클래스 불균형 여부 → 평가 지표 선택에 영향 |

### 1-4. 수치형 vs. 범주형 특성

| 구분 | 수치형 (Numerical) | 범주형 (Categorical) |
|------|-------------------|---------------------|
| 정의 | 숫자로 표현되는 연속/이산 값 | 카테고리/그룹을 나타내는 값 |
| 예시 | 나이, 매출, DAU, 가격 | 성별, 지역, 요금제, 등급 |
| 연산 | 평균, 표준편차 등 수학 연산 가능 | 수학 연산 의미 없음 (최빈값만 의미) |
| 전처리 | 스케일링 (StandardScaler 등) | 인코딩 (OneHot, Label 등) |
| describe() | count, mean, std, min, max | count, unique, top, freq |

### 1-5. 클래스 불균형 (Imbalanced Classes)
분류 문제에서 클래스별 샘플 수가 크게 다른 현상
ML 모델 성능에 큰 영향을 미친다.

| 균형 상태 | 비율 예시 | 영향 |
|-----------|----------|------|
| 균형 | 양성 50% : 음성 50% | 대부분의 알고리즘이 잘 작동 |
| 약간 불균형 | 양성 30% : 음성 70% | 주의 필요, stratify 사용 |
| 심한 불균형 | 양성 2% : 음성 98% | Accuracy 무의미, F1/ROC-AUC 사용 필수 |

**PM/분석 예시**
> 이탈 예측에서 이탈 고객 5%, 유지 고객 95%라면 → 모델이 무조건 "유지"라고만 예측해도 정확도 95%가 된다.
> 이런 경우 Accuracy는 무의미하고, Precision/Recall/F1이 중요하다.
