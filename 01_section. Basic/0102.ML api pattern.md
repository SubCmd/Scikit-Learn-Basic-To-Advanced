# Phase 1-2: Scikit-Learn API 패턴 (Estimator / Transfomer)

## 1. 개념 및 정의

### 1-1. 왜 API 패턴이 중요한가?
: scikit-learn의 가장 큰 장점은 모든 알고리즘이 동일한 인터페이스를 따른다
KNN이든 Random Forest이든 XGBoost든, `fit()` → `predict()` 패턴은 변하지 않는다.
이 패턴만 익히면 수십 개 알고리즘을 동일한 방식으로 사용할 수 있다.

---

### 1-2. 3가지 핵심 패턴
scikit-learn 객체 유형
├── Estimator (추정기)     → fit() + predict()     : 예측 모델
├── Transformer (변환기)   → fit() + transform()   : 데이터 변환
└── Pipeline (파이프라인)  → 위 두 가지를 체이닝     : 워크플로우 통합

---

### 1-3. Estimator 패턴 (예측 모델)
모든 지도학습 모델이 따르는 패턴

model = SomeModel(하이퍼파라미터)    # 1단계: 모델 객체 생성
model.fit(X_train, y_train)         # 2단계: 학습 (파라미터 추정)
y_pred = model.predict(X_test)      # 3단계: 예측
score = model.score(X_test, y_test) # 4단계: 평가

 메서드 | 역할 | 비유 |
|--------|------|------|
| `__init__()` | 하이퍼파라미터 설정 | 오븐 온도, 시간 설정 |
| `fit(X, y)` | 데이터로부터 패턴(파라미터) 학습 | 레시피대로 요리 시작 |
| `predict(X)` | 새로운 데이터에 대한 예측 | 새 재료로 같은 요리 만들기 |
| `score(X, y)` | 예측 성능 평가 | 요리 맛 평가 |
| `get_params()` | 현재 하이퍼파라미터 확인 | 설정값 확인 |

#### 추가 메서드
| 메서드 | 역할 | 사용 예시 |
|--------|------|-----------|
| `predict_proba(X)` | 각 클래스에 속할 **확률** 반환 | 이탈 확률 70%, 유지 확률 30% |
| `decision_function(X)` | 결정 경계까지의 거리 반환 | SVM에서 마진 거리 확인 |
 

> **PM 관점**: `predict()`는 "이탈할 것이다/아니다"를 알려주고, `predict_proba()`는 "이탈 확률이 73%다"를 알려준다. 비즈니스 의사결정에는 확률이 더 유용한 경우가 많다 (예: 이탈 확률 상위 10% 고객에게 쿠폰 발송).

### 1-4. Transformer 패턴 (데이터 변환)
데이터 전처리 도구가 따르는 패턴

```
scaler = SomeTransformer()                        # 1단계: 변환기 생성
scaler.fit(X_train)                               # 2단계: 훈련 데이터에서 통계값 학습
X_train_transformed = scaler.transform(X_train)   # 3단계: 훈련 데이터 변환
X_test_transformed = scaler.transform(X_test)     # 4단계: 테스트 데이터 변환
 
# fit + transform 동시 실행 (훈련 데이터에만!)
X_train_transformed = scaler.fit_transform(X_train)
```

| 메서드 | 역할 | 비유 |
|--------|------|------|
| `fit(X)` | 변환에 필요한 통계값(평균, 표준편차 등) 학습 | 자(ruler)의 눈금을 데이터에 맞춰 만드는 것 |
| `transform(X)` | 학습된 통계값으로 데이터 변환 | 만들어진 자로 실제 측정하는 것 |
| `fit_transform(X)` | fit + transform 동시 실행 | 자를 만들면서 동시에 측정 |
| `inverse_transform(X)` | 변환을 되돌리기 | 변환된 값을 원래 스케일로 복원 |


#### ⚠️ 가장 중요한 원칙: Data Leakage 방지
```
✅ 올바른 순서:
scaler.fit(X_train)              # 훈련 데이터에만 fit
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)   # 테스트는 transform만!
 
❌ 잘못된 순서:
scaler.fit(X_전체)               # 전체 데이터에 fit → Data Leakage!
scaler.fit(X_test)               # 테스트 데이터에 fit → Data Leakage!
scaler.fit_transform(X_test)     # 테스트에 fit_transform → Data Leakage!
```

| 용어 | 설명 |
|------|------|
| **Data Leakage (데이터 누출)** | 테스트 데이터의 정보가 모델 학습/전처리 과정에 유출되는 것. 모델이 "미래를 엿본" 상태가 되어 실제보다 성능이 부풀려진다. |
 
**비유**: 시험 문제를 미리 본 학생의 성적은 신뢰할 수 없다. `fit()`은 "문제를 보는 행위"이므로, 훈련 데이터에서만 수행해야 한다.

---

### 1-5. Estimator vs. Transformer 비교

| 구분 | Estimator (예측기) | Transformer (변환기) |
|------|-------------------|---------------------|
| 목적 | **예측** (y값을 맞추기) | **변환** (X를 가공) |
| 핵심 메서드 | `fit()` + `predict()` | `fit()` + `transform()` |
| fit()의 의미 | X와 y의 관계를 학습 | X의 통계적 속성을 학습 |
| 입력 | `fit(X, y)` — y 필요 | `fit(X)` — y 불필요 |
| 출력 | 예측값 (y_pred) | 변환된 X |
| 예시 | KNN, 로지스틱 회귀, SVM | StandardScaler, PCA, OneHotEncoder |

---

### 1-6. 학습된 속성 확인 (tralling underscore 규칙)
scikit-learn에서 학습을 통해 결정된 속성은 이름 끝에 밑줄(`_`)이 붙는다.

```python
# Estimator 학습 후
model.coef_           # 학습된 회귀 계수
model.intercept_      # 학습된 절편
model.classes_        # 학습된 클래스 목록
model.n_features_in_  # 학습에 사용된 특성 수

# Transformer 학습 후
scaler.mean_          # 학습된 평균값
scaler.scale_         # 학습된 표준편차
scaler.n_features_in_ # 학습에 사용된 특성 수
```

> **규칙**: 밑줄(`_`)이 붙은 속성은 `fit()` 이후에만 접근 가능하다. `fit()` 전에 접근하면 `NotFittedError`가 발생한다.
