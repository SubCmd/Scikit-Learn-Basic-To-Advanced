# Phase 1-1: 머신러닝 개요 & Scikit-Learn 소개

## 1. 개념 및 정의

### 1-1. 머신러닝이란?
: 머신러닝(Machine Learning)은 명시적으로 프로그래밍하지 않고, 데이터에서 패턴을 학습하여 예측이나 결정을 내리는 알고리즘

- 전통적 프로그래밍 vs. 머신러닝
| 구분 | 전통적 프로그래밍 | 머신러닝 |
|------|-------------------|----------|
| 입력 | 데이터 + **규칙** | 데이터 + **결과(정답)** |
| 출력 | **결과** | **규칙(모델)** |
| 비유 | 레시피를 보고 요리하기 | 완성된 요리를 보고 레시피를 역추론하기 |
| PM 비유 | "DAU가 10만 이하면 알림 보내라"고 직접 코딩 | 과거 DAU 데이터와 이탈률을 학습시켜 이탈 예측 모델을 만듦 |

- 왜 머신러닝이 필요한가? (PM/데이터 분석가 관점)
1) 규칙이 너무 복잡할 때: 고객 이탈 예측 - 어떤 조건이 이탈을 유발하는지 수백 개의 규칙을 사람이 정하기 어려움
2) 규칙이 계속 변할 때: 스팸 필터 - 스패머가 패턴을 바꾸면 규칙도 매번 수정해야 함.
3) 인간이 인식하기 어려운 패턴: 추천 시스템 - 수백만 명의 행동 패턴에서 개인화된 추천을 만드는 것.

### 1-2. 머신러닝의 3가지 유형

머신러닝
├── 지도학습 (Supervised Learning) ← scikit-learn의 핵심
│   ├── 분류 (Classification): 카테고리 예측
│   └── 회귀 (Regression): 연속 값 예측
├── 비지도학습 (Unsupervised Learning)
│   ├── 군집화 (Clustering): 유사 그룹 발견
│   └── 차원 축소 (Dimensionality Reduction)
└── 강화학습 (Reinforcement Learning) ← scikit-learn에서 미지원


#### 비교표
| 유형 | 정답(Label) 유무 | 목표 | PM/분석가 사용 예시 |
|------|------------------|------|---------------------|
| **지도학습 - 분류** | ✅ 있음 (범주형) | 카테고리 예측 | 고객 이탈 여부(이탈/유지), 스팸 필터, 리뷰 감성 분류 |
| **지도학습 - 회귀** | ✅ 있음 (연속형) | 수치 예측 | LTV 예측, 매출 예측, DAU 예측 |
| **비지도학습 - 군집화** | ❌ 없음 | 그룹 발견 | RFM 기반 고객 세그먼트, 유사 상품 그룹핑 |
| **비지도학습 - 차원축소** | ❌ 없음 | 특성 압축 | 고차원 데이터 시각화, 노이즈 제거 |
| **강화학습** | ❌ (보상 신호) | 보상 최대화 | 동적 가격 책정, 추천 시스템 최적화 |

- 비유로 이해하기 
> 지도학습: 선생님이 문제와 정답을 함께 주고 공부시킨 뒤, 새로운 문제를 풀게 하는 것
> 비지도학습: 정답 없이 학생들 스스로 비슷한 것끼리 묶어보라고 하는 것
> 강화학습: 게임을 직접 플레이하면서 점수(보상)을 높이는 전략을 스스로 터득하는 것

### 1-3. Scikit-Learn이란?
Python 가장 대표적인 머신러닝 라이브러리

#### 핵심 특징
| 특징 | 설명 |
|------|------|
| **일관된 API** | 모든 알고리즘이 `fit()` → `predict()` / `transform()` 동일 패턴 |
| **풍부한 내장 데이터셋** | Iris, California Housing 등 학습용 데이터 바로 사용 가능 |
| **전처리 도구** | 스케일링, 인코딩, 결측치 처리 등 전처리 파이프라인 내장 |
| **모델 평가 도구** | 교차 검증, GridSearch, 다양한 평가 지표 내장 |
| **우수한 문서화** | 공식 문서가 튜토리얼 수준으로 상세 |

### ML 생태계에서 scikit-learn의 위치?
데이터 수집/정제          모델링                    서빙/배포
─────────────────    ─────────────────────    ──────────────
pandas, numpy   →    scikit-learn (전통 ML)  →  Flask, FastAPI
SQL, Spark      →    XGBoost, LightGBM       →  MLflow
                →    PyTorch, TensorFlow (DL) →  TorchServe

- scikit-learn은 전통적 머신러닝의 표준이며, XGBosst/LightGBM도 scikit-learn의 API 패턴을 따름
  즉, scikit-learn을 배우면 다른 ML 라이브러리도 동일한 방식을 사용 가능

### 1-4. 핵심 용어 정리

| 용어 | 영문 | 설명 | PM/분석 비유 |
|------|------|------|-------------|
| 특성 (피처) | Feature | 모델에 입력되는 독립 변수 (X) | GA4의 이벤트 파라미터, 사용자 속성 |
| 레이블/타겟 | Label/Target | 예측하려는 종속 변수 (y) | 이탈 여부, 구매 금액 |
| 샘플 | Sample | 하나의 데이터 행 | 한 명의 유저 데이터 |
| 훈련 세트 | Training Set | 모델 학습에 사용하는 데이터 | 과거 3개월 유저 데이터로 학습 |
| 테스트 세트 | Test Set | 모델 성능 평가에 사용하는 데이터 | 최근 1개월 데이터로 검증 |
| 과적합 | Overfitting | 훈련 데이터에만 지나치게 잘 맞는 현상 | "시험 기출만 달달 외워서 새 문제를 못 푸는 것" |
| 과소적합 | Underfitting | 훈련 데이터도 제대로 학습하지 못하는 현상 | "기출도 못 외운 상태" |
| 하이퍼파라미터 | Hyperparameter | 사용자가 직접 설정하는 모델의 외부 설정값 | A/B 테스트에서 실험 조건을 직접 설정하는 것과 유사 |
| 파라미터 | Parameter | 모델이 데이터로부터 **자동 학습**하는 내부 값 | 모델이 알아서 찾아낸 패턴/가중치 |

#### 하이퍼파라미터 vs. 파라미터 (자주 혼동!)
| 구분 | 하이퍼파라미터 | 파라미터 |
|------|---------------|---------|
| 누가 설정? | **사람**이 직접 설정 | **모델**이 학습으로 결정 |
| 언제 설정? | 학습 **전**에 설정 | 학습 **중**에 결정 |
| 예시 | KNN의 K값, 학습률, 트리 깊이 | 회귀 계수(가중치), 결정 트리의 분할 기준 |
| 비유 | 요리 전 오븐 온도를 몇 도로 설정할지 | 오븐이 자동으로 최적 굽기를 찾아내는 것 |
