# Phase 1-5: 특성 스케일링 (Feature Scaling)

## 1. 개념 및 정의

### 1-1. 왜 스케일링이 필요한가?
특성들의 단위와 범위가 다르면 일부 알고리즘이 제대로 작동하지 않음

```
특성 A: 나이 (20 ~ 60)
특성 B: 연봉 (3000만 ~ 1억 2000만)

→ 거리 기반 알고리즘(KNN)에서는 연봉의 차이가 나이의 차이를 압도
→ 모델이 "연봉만 판단"하는 편향이 발생
```

| 비유 | 설명 |
|------|------|
| **자(ruler)** | cm로 재는 키와 kg으로 재는 몸무게를 같은 자로 비교할 수 없다. 같은 단위(같은 자)로 맞춰야 공정한 비교가 가능하다. |
| **시험 점수** | 수학(100점 만점)과 영어(990점 만점)를 단순 합산하면 영어가 압도한다. 각각 100점 만점으로 환산해야 공정하다. |

---

### 1-2. 스케일링이 필요한/불필요한 알고리즘

| 스케일링 필요 ✅ | 스케일링 불필요 ❌ |
|-----------------|------------------|
| KNN (거리 기반) | 결정 트리 (Decision Tree) |
| SVM (마진 최대화) | 랜덤 포레스트 (Random Forest) |
| 로지스틱 회귀 (경사 하강법) | XGBoost / LightGBM |
| 선형 회귀 (규제 적용 시) | 나이브 베이즈 |
| 신경망 / 딥러닝 | |
| PCA (분산 기반) | |
| K-Means (거리 기반) | |

**원칙**: **거리**를 계산하거나 **경사 하강법**을 사용하는 알고리즘은 스케일링 필수. **트리 기반** 알고리즘은 분할 기준만 보므로 스케일 무관.

---

### 1-3. 3대 스케일러 비교

| 스케일러 | 공식 | 변환 후 범위 | 특징 | 사용 시점 |
|----------|------|-------------|------|-----------|
| **StandardScaler** | `z = (x - mean) / std` | 평균 0, 표준편차 1 | 가장 범용적 | 대부분의 상황 (기본 선택) |
| **MinMaxScaler** | `x' = (x - min) / (max - min)` | 0 ~ 1 | 범위가 명확 | 신경망, 이미지 데이터 |
| **RobustScaler** | `x' = (x - median) / IQR` | 중앙값 0 | 이상치에 강건 | **이상치가 많은 데이터** |

---

### 1-4. 이상치가 스케일링에 미치는 영향
 
```
데이터: [10, 20, 30, 40, 50, 1000]  ← 1000이 이상치
 
StandardScaler:
  mean = 191.67, std = 362.84
  → 대부분의 정상 데이터가 음수 근처로 몰림 (비효율적)
 
MinMaxScaler:
  min = 10, max = 1000
  → 정상 데이터는 0~0.04 범위에 몰림 (대부분 0 근처)
 
RobustScaler:
  median = 35, IQR = 25
  → 이상치의 영향을 덜 받음 (정상 데이터가 고르게 분포)
```

| 상황 | 추천 스케일러 |
|------|-------------|
| 정규 분포에 가까운 데이터 | StandardScaler |
| 범위가 명확해야 하는 경우 | MinMaxScaler |
| 이상치가 있는 데이터 | **RobustScaler** |
| 어떤 스케일러를 쓸지 모르겠을 때 | **StandardScaler** (기본 선택) |

---

### 1-5. 스케일링 순서 원칙 (Data Leakage 방지)

```
✅ 올바른 순서:
1. train_test_split → 훈련/테스트 분리
2. scaler.fit(X_train) → 훈련 데이터에만 fit
3. scaler.transform(X_train) → 훈련 데이터 변환
4. scaler.transform(X_test) → 테스트 데이터 변환 (같은 기준!)
 
❌ 잘못된 순서:
1. scaler.fit(X_전체) → 전체 데이터에 fit (Data Leakage!)
2. scaler.transform(X_전체) → 전체 변환
3. train_test_split → 변환된 데이터 분리
```
 
> **핵심**: 항상 **분리 먼저, 스케일링 나중**. 스케일링의 fit은 **훈련 데이터에만**.
