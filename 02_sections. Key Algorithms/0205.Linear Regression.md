# Phase 2-5: Linear Regression (선형 회귀 & 규제 회귀 : Ridge / Lasso / ElasticNet)

---

## 1. 개념 및 정의

### 1-1. 선형 회귀란?
특성(X)과 타겟(y) 간의 선형 관계를 찾아 연속 값을 예측하는 알고리즘

```
y = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
 
y: 예측값 (주택 가격, 매출 등)
w: 가중치/계수 (각 특성의 기여도)
x: 특성값
b: 절편 (bias)
```

##### 비유
> "각 특성에 점수(가중치)를 매겨서 총점을 계산하는 것"
> - 주택가격 = 0.5 x 면적 + 0.3 x 역세권거리 + 0.1 x 층수 + 절편

---

### 1-2. 회귀 평가 지표

| 지표 | 공식 개요 | 범위 | 해석 | PM 비유 | 
|------|----------|------|------|---------|
| **MAE(Mean Absolute Error)** | 평균 절대 오차 | 0 ~ ∞ | 예측과 실제의 평균 차이 | "평균적으로 얼마나 빗나갔나" |
| **MSE(Mean Square Error)** | 평균 제곱 오차 | 0 ~ ∞ | 큰 오차에 더 큰 패널티 | "큰 실수를 중요하게 봄" |
| **RMSE(Root Mean Square Error)** | √MSE | 0 ~ ∞ | 원래 단위로 해석 가능 | "실제 단위로 얼마나 틀렸나" |
| **R²(R-squared)** | 결정 계수 | -∞ ~ 1 | 모델이 설명하는 분산 비율 | "모델이 얼마나 잘 맞추는가" |


#### R² 해석 가이드
 
| R² 값 | 의미 |
|--------|------|
| 1.0 | 완벽한 예측 |
| 0.8~1.0 | 매우 좋음 |
| 0.5~0.8 | 보통 |
| 0.0 | 평균으로만 예측하는 것과 동일 |
| < 0 | 평균보다 못한 모델 (매우 나쁨) |

---

### 1-3. 과적합 문제와 규제의 필요성
선형 회귀는 특성이 많거나 다중공선성이 있으면 과적합될 수 있음

```
과적합 상황:
- 특성 100개, 샘플 200개
- 일부 계수가 극단적으로 커짐 (w = 50000)
- 훈련 데이터에만 과도하게 맞춤

해결: 규제(Regularization) - 계수의 크기를 제한
```

### 1-4. 규제 회귀 3종 비교

| 모델 | 규제 유형 | 패널티 | 특징 | 사용 시점 |
| ---- | -------- | ----- | ---- | -------- |
| Ridge | L2 | α × Σ(wᵢ²) | 계수를 작게 줄임. 0으로 만들지 않음 | 모든 특성이 유용할 때 |
| Lasso | L1 | α × Σ|wᵢ| | 불필요한 특성 계수를 0으로 만듦 | 특성 선택이 필요할 때 | 
| ElasticNet | L1+L2 | α₁×Σ|wᵢ| + α₂×Σ(wᵢ²) | 두 규제의 장점 결합 | 상관된 특성이 많은 때 |

#### 비유

| 모델 | 비유 |
| ---- | ---- | 
| LinearRegression | 제한 없이 모든 변수를 사용하는 모델 | 
| Ridge | "모든 직원을 유지하되, 과도한 보너스는 줄여라" | 
| Lasso | "불필요한 직원은 해고하고(계수=0), 나머지만 남겨라" |
| ElasticNet | "일부 해고 + 나머지 보너스 조정" |

---

### 1-5. alpha (α) 파라미터

모든 규제 회귀에서 alpha는 규제 강조를 조절함.

| alpha | 규제 강조 | 모델 | 위험 | 
| ----- | -------- | ---- | ---- | 
| 0 | 규제 없음 | =LinearRegression | 과적합 | 
| 작은 값 (0.01) | 약한 규제 | 약간 제한된 선형 회귀 | - | 
| 1.0 (기본) | 보통 | 적절한 균형 | - | 
| 큰 값 (100) | 강한 규제 | 매우 단순한 모델 | 과소적합 | 

> 로지스틱 회귀의 C와 반대: C가 크면 규제 약함, alpha가 크면 규제 강함

### 1-6. Lasso의 특성 선택 효과

```
특성 10개, alpha=0.1로 Lasso 학습 후:

feature_1:  0.523  ← 중요한 특성 (유지)
feature_2:  0.000  ← 불필요 (제거됨)
feature_3: -0.312  ← 중요한 특성 (유지)
feature_4:  0.000  ← 불필요 (제거됨)
...

→ 자동으로 불필요한 특성을 걸러냄 (Feature Selection)
→ PM 관점: "100개 특성 중 실제 중요한 15개만 남겼습니다"
```

### 전체 요약

| 핵심 개념 | 설명 |
|-----------|------|
| **선형 회귀** | y = w₁x₁ + ... + wₙxₙ + b, 연속 값 예측 |
| **평가 지표** | MAE, RMSE(단위 해석), R²(설명력) |
| **Ridge (L2)** | 계수를 줄이되 0으로 만들지 않음 → 모든 특성 유지 |
| **Lasso (L1)** | 불필요한 계수를 0으로 → **자동 특성 선택** |
| **ElasticNet** | L1 + L2 혼합 → 상관 특성이 많을 때 |
| **alpha** | 규제 강도 — 크면 강한 규제(단순), 작으면 약한 규제(복잡) |
| **RidgeCV / LassoCV** | 내장 교차 검증으로 최적 alpha 자동 탐색 |
