# Phase 2-2: KNN (K-최근접 이웃, K-Nearest Neighbors)

## 1. 개념 및 정의

### 1-1. KNN이란?
새로운 데이터가 들어오면, 가장 가까운 K개의 이웃을 찾아 다수결(분류) 또는 평균(회귀)으로 예측하는 알고리즘. 별도 학습 과정이 없는 게으른 학습(Lazy Learning) 방식

> Lazy Learning : 훈련 과정에서 일반화된 모델을 미리 만들지 않고, 새로운 입력 데이터가 들어올 때까지 학습(계산)을 미루는 방식

#### 비유
> "내 주변 친구 5명 중 4명이 A반이면, 나도 A반일 가능성이 높다."

| 특징 | 설명 |
|------|------|
| **학습 방식** | 게으른 학습 — fit() 시 데이터를 저장만 함 |
| **예측 방식** | 새 데이터와 모든 훈련 데이터의 거리를 계산 |
| **분류** | K개 이웃 중 다수결 투표 |
| **회귀** | K개 이웃의 타겟 평균값 |

---

### 1-2. 거리 측정 방법

| 거리 | 공식 | 특징 | 비유 |
|------|------|------|------|
| **유클리드 (Euclidean)** | √(Σ(x-y)²) | 직선 거리, 기본값 | 새가 날아가는 최단 거리 |
| **맨해튼 (Manhattan)** | Σ\|x-y\| | 격자 거리 | 도시 블록을 따라 걷는 거리 |
| **민코프스키 (Minkowski)** | (Σ\|x-y\|^p)^(1/p) | p=2이면 유클리드, p=1이면 맨해튼 | 일반화된 거리 |
 
> **실무 팁**: 대부분의 경우 기본값(유클리드)으로 충분하다.

---

### 1-3. K값의 영향

| K값 | 경계 | 특징 | 위험 |
|-----|------|------|------|
| **K가 작을 때** (K=1) | 복잡한 경계 | 노이즈에 민감 | **과적합** |
| **K가 클 때** (K=50) | 단순한 경계 | 세부 패턴 놓침 | **과소적합** |
| **최적 K** | 적절한 균형 | 교차 검증으로 탐색 | |
 
> **K는 홀수를 사용**: 이진 분류에서 동점을 방지하기 위해 홀수 권장 (3, 5, 7, 9...)

---

### 1-4. KNN의 장단점

| 장점 | 단점 |
|------|------|
| 직관적이고 구현이 단순 | **느림** — 예측 시 모든 데이터와 거리 계산 |
| 학습 과정이 없음 (빠른 fit) | 대규모 데이터에서 비효율적 |
| 비선형 경계 학습 가능 | **스케일링 필수** (거리 기반) |
| 하이퍼파라미터가 적음 | 고차원 데이터에서 성능 저하 (차원의 저주) |


#### 차원의 저주 (Curse of Dimension)
특성 수가 많아질수록, 모든 점 사이의 거리가 비슷해져서 "가까운 이웃"의 의미가 퇴색됨.

| 특성 수 | KNN 성능 | 이유 |
|---------|---------|------|
| 2~10개 | 좋음 | 거리가 의미 있음 |
| 10~50개 | 보통 | 차원 증가로 거리 차이 줄어듦 |
| 50개 이상 | 나쁨 | 차원의 저주 → PCA로 차원 축소 권장 |

### 1-5. weights 파라미터

| 값 | 설명 | 비유 |
|----|------|------|
| `'uniform'` (기본) | 모든 이웃에 동일한 가중치 | 친구 5명의 의견이 모두 동등 |
| `'distance'` | 가까운 이웃일수록 높은 가중치 | 가까운 친구의 의견을 더 신뢰 |
