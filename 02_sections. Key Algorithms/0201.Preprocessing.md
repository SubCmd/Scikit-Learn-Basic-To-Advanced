# Phase 2-1: 결측치 처리 & 범주형 인코딩

## 1. 개념 및 정의

### 1-1. 왜 전처리가 필요한가?
실제 데이터는 scikit-learn의 내장 데이터셋처럼 깨끗하지 않음.
결측치(NaN), 범주형 데이터(문자열) 등이 포함되어 있으며,
scikit-learn 모델은 **숫자 행렬(numpy array)**만 입력으로 받을 수 있음.

```
실제 데이터의 문제들:
├── 결측치 (NaN, null)     → 모델이 처리 불가
├── 범주형 데이터 (문자열)  → 모델이 처리 불가
├── 이상치 (Outlier)       → 모델 성능 저하
└── 스케일 차이            → Phase 1-5에서 학습 완료 ✅
```


### 1-2. 결측치 처리 전략

| 전략 | 방법 | 장점 | 단점 | 사용 시점 |
|------|------|------|------|-----------|
| **삭제** | 해당 행/열 제거 | 단순, 빠름 | 데이터 손실 | 결측 비율 < 5% |
| **단순 대체** | 평균/중앙값/최빈값 | 데이터 보존 | 분포 왜곡 가능 | 일반적 상황 |
| **KNN 대체** | 유사 샘플로 추정 | 더 정교한 추정 | 느림, 스케일 민감 | 결측 패턴이 복잡할 때 |
| **상수 대체** | 특정 값(-999 등) | 결측 자체를 특성화 | 트리 모델에만 유효 | 결측 여부 자체가 의미 있을 때 |


#### 수치형 vs. 범주형 결측치 처리
| 데이터 타입 | 추천 대체 전략 | 비유 |
|------------|---------------|------|
| 수치형 (나이, 매출) | **중앙값(median)** — 이상치에 강건 | 시험 성적 결측 → 반 중앙값으로 채움 |
| 범주형 (성별, 지역) | **최빈값(most_frequent)** | 결측된 지역 → 가장 많은 지역으로 채움 |

> **PM 관점**: 결측치 처리 전략은 비즈니스 맥락에 따라 달라진다.
> 예를 들어, 고객 연령이 없으면 중앙값으로 채우는 것이 합리적이지만, 구매 금액이 없으면 0으로 채우는 것이 더 맞을 수 있다.


### 1-3. 범주형 인코딩 전략
scikit-learn 모델은 문자열을 입력받지 못하므로, 범주형 데이터를 숫자로 변환해야 함.

| 인코딩 | 방법 | 사용 대상 | 예시 |
|--------|------|-----------|------|
| **LabelEncoder** | 문자→정수 매핑 | **타겟(y)에만** | cat→0, dog→1, fish→2 |
| **OrdinalEncoder** | 순서 있는 범주→정수 | 순서형 특성(X) | low→0, medium→1, high→2 |
| **OneHotEncoder** | 범주를 이진 벡터로 | **순서 없는 특성(X)** | red→[1,0,0], blue→[0,1,0] |

#### 왜 순서 없는 범주에 정수 인코딩을 쓰면 안 되는가?
```
성별: 남성→0, 여성→1
→ 모델이 "여성(1) > 남성(0)" 이라는 순서 관계를 학습할 수 있음
→ 실제로는 순서가 없으므로 잘못된 학습
 
해결: OneHotEncoding
남성 → [1, 0]
여성 → [0, 1]
→ 독립적인 이진 특성으로 변환 → 순서 관계 없음
```

#### OneHotEncoder의 drop='first' 옵션

| 옵션 | 변환 결과 | 설명 |
|------|-----------|------|
| `drop=None` | red→[1,0,0], blue→[0,1,0], green→[0,0,1] | 3개 컬럼 생성 |
| `drop='first'` | red→[0,0], blue→[1,0], green→[0,1] | **2개 컬럼** (다중공선성 방지) |
 
> `drop='first'`를 쓰면, 첫 번째 범주(red)는 나머지가 모두 0일 때를 의미한다. 선형 모델에서 다중공선성을 방지하는 데 유용하다.

> 다중공선성(Multicolinearity)란?
> 통계학 회귀분석에서 독립변수들 간에 강한 상관관계가 나타나는 문제


### 1-4. ColumnTransformer - 수치형/범주형 동시 처리
실제 데이터에는 수치형과 범주형 특성이 섞여있음.
`ColumnTransformer`를 사용하면 각 특성 유형에 다른 전처리 적용 가능

```
전체 데이터
├── 수치형 특성 (age, income) → SimpleImputer(median) → StandardScaler
└── 범주형 특성 (gender, city) → SimpleImputer(most_frequent) → OneHotEncoder
         ↓
    ColumnTransformer로 합치기
         ↓
    하나의 전처리된 행렬 (numpy array)
```
