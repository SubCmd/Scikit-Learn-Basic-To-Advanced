# Phase 2-4: 결정 트리 (Decision Tree)

## 1. 개념 및 정의

### 1-1. 결정 트리란?
특성의 조건으로 데이터를 반복 분할하여 트리 구조의 규칙을 만드는 알고리즘
각 노드에서 "어떤 특성을, 어떤 기준으로 나눌 것인가?"를 결정
> 한 번 분할할 때마다 target 고유값들이 최대한 한 그룹에 깔끔하게 모이도록(순도: High / 불순도: Low) 영역을 쪼개나가는 알고리즘

비유
> "스무고개 게임 - 질문을 통해 답을 좁혀나가는 과정"

```
Q. petal length ≤ 2.45?
├── Yes → setosa (50개, 100% 순수)
└── No
    Q. petal width ≤ 1.75?
    ├── Yes → versicolor (대부분)
    └── No → virginica (대부분)
```

---

### 1-2. 트리 구조 용어

| 용어 | 설명 | 비유 |
|------|------|------|
| **루트 노드** | 가장 위의 첫 번째 분할 | 첫 번째 질문 |
| **내부 노드** | 중간 분할 | 후속 질문 |
| **리프 노드** | 최종 결과 (더 이상 분할 안 함) | 최종 답 |
| **깊이(Depth)** | 루트에서 리프까지의 거리 | 질문 횟수 |
| **분할 기준** | 노드에서 데이터를 나누는 조건 | "나이 ≥ 30?" |
 
---

### 1-3. 분할 기준: 불순도(Impurity)

| 지표 | 공식 | 범위 | 설명 | 
|------|------|------|------|
| **Gini 불순도** | 1 - Σ(pᵢ²) | 0 ~ 0.5 (이진) | 기본값, 빠름 |
| **엔트로피** | -Σ(pᵢ × log₂(pᵢ)) | 0 ~ 1 (이진) | 정보이론 기반 |

| 불순도 값 | 의미 |
|-----------|------|
| **0** | 완벽하게 순수 — 한 클래스만 존재 |
| **최대(50:50)** | 완벽하게 혼합 — 모든 클래스가 동일 비율 |

> **실무**
> Gini와 Entropy의 성능 차이는 거의 없음
> 기본값(Gini)을 사용해도 무방

### 1-4. 과적합과 가지치기 (Pruing)
결정 트리는 **제한 없이 성장시키면 과적합**이 발생.

```
과적합 트리:
- 깊이 20, 리프 노드 500개
- 훈련 정확도: 100% ✅
- 테스트 정확도: 72% ❌
- 각 리프에 샘플 1~2개 → 노이즈까지 외움

가지치기된 트리:
- 깊이 5, 리프 노드 20개
- 훈련 정확도: 95%
- 테스트 정확도: 90% ✅
- 일반화된 규칙 학습
```

#### 가지치기 하이퍼파라미터

| 파라미터 | 설명 | 작게 설정하면 | 크게 설정하면 |
|----------|------|-------------|-------------|
| `max_depth` | 트리 최대 깊이 | 단순 모델 (과소적합) | 복잡 모델 (과적합) |
| `min_samples_split` | 분할을 위한 최소 샘플 수 | 더 많이 분할 | 분할 억제 |
| `min_samples_leaf` | 리프 노드 최소 샘플 수 | 작은 리프 허용 | 큰 리프만 허용 |
| `max_features` | 분할 시 고려할 최대 특성 수 | 다양성 증가 | 최선 특성 선택 |
| `max_leaf_nodes` | 최대 리프 노드 수 | — | 리프 수 제한 |

### 1-5. 결정 트리의 장단점

| 장점 | 단점 | 
|------|------|
| **해석이 쉬움** (규칙 추출 가능) | **과적합에 취약** |
| **스케일링 불필요** | 불안정 (데이터 변화에 민감) |
| **특성 중요도 제공** | 선형 관계 학습에 비효율 |
| 수치형 + 범주형 모두 처리 | 단독 사용 시 성능 제한적 |
| 빠른 학습/예측 | (→ 앙상블로 극복: 랜덤 포레스트) |

---

### 1-6. 특성 중요도 (Feature Importance)
결정 트리는 각 특성이 불순도 감소에 **얼마나 기여했는지**를 자동으로 계산함.

> **PM 가치**
> "이탈 예측에서 가장 중요한 변수는 '최근 접속일'이고, 두 번째는 '고객센터 문의 횟수'입니다."라는 비즈니스 인사이트 도출 가능
