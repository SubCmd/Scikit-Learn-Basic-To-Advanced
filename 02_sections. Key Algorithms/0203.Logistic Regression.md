## 1. 개념 및 정의

### 1-1. 로지스틱 회귀란?
이름에 "회귀"가 있지만 분류 알고리즘
선형 결합 결괄르 시그모이드(Sigmoid) 함수로 0~1 확률로 변환하여 분류

```
입력(특성) →  선형 결합(가중합)   → 시그모이드 함수     → 확률(0~1)  → 분류
    X      z = w₁x₁ + w₂x₂ + b   σ(z) = 1/(1+e^(-z))     P≥0.5    → 1
                                                         P<0.5    → 0
```

비유
> "각 특성에 점수(가중치)를 매겨 총점을 계산한 뒤, 합격선(0.5)을 넘으면 합격, 아니면 불합격

---

### 1-2. 시그모이드 함수

| 입력(z) | 출력 σ(z) | 해석 |
|---------|-----------|------|
| z → -∞ | → 0 | 양성일 확률 거의 0% |
| z = 0 | 0.5 | 반반 (결정 경계) |
| z → +∞ | → 1 | 양성일 확률 거의 100% |
 
핵심: 어떤 값이든 0~1 사이로 변환 → **확률로 해석 가능**

### 1-3. 로지스틱 회귀 vs. 다른 분류 모델

| 특성 | 로지스틱 회귀 | KNN | 결정 트리 |
|------|-------------|-----|-----------|
| 결정 경계 | **선형** | 비선형 | 비선형 (축 평행) |
| 해석 가능성 | **높음** (계수 해석) | 낮음 | 중간 (규칙 추출) |
| 확률 출력 | **자연스럽게 확률 제공** | 이웃 비율로 추정 | 리프 노드 비율 |
| 스케일링 | 필요 (수렴 속도) | 필수 (거리) | 불필요 |
| 학습 속도 | 빠름 | fit 없음 (predict 느림) | 빠름 |
| PM 활용 | **변수 기여도 분석에 최적** | 유사 고객 찾기 | 의사결정 규칙 도출 |

### 1-4. 규제 파라미터 C
로지스틱 회귀에서 C는 규제 강조를 조절함.

| C값 | 규제 강조 | 모델 복잡도 | 위험 |
| --- | -------- | ---------- | ---- |
| C=0.01(작음) | 강한 규제 | 단순한 모델 | 과소적합 |
| C=1.0(기본) | 보통 | 적절한 균형 | |
| C=100(큼) | 약한 규제 | 복잡한 모델 | 과적합 |

> 비유
> C가 크면 "데이터를 최대한 맞추려고 노력"
> C가 작으면 "좀 틀려도 괜찮으니 단순하게" -> 모델 일반화

---

### 1-5. 다중 클래스 분류
로지스틱 회귀는 기본적으로 이진 분류이지만, 다중 클래스도 지원

| 전략 | 파라미터 | 방식 |
|------|----------|------|
| **OvR** (One-vs-Rest) | `multi_class='ovr'` | 각 클래스마다 "이 클래스 vs 나머지" 이진 분류 || **Multinomial** | `multi_class='multinomial'` | 소프트맥스로 모든 클래스 동시 분류 |
| **Auto** (기본) | `multi_class='auto'` | 2클래스면 이진, 3+ 클래스면 multinomial |

### 1.6 로지스틱 회귀의 해석력 (PM에게 가장 중요)

```python
# 이탈 예측 모델의 계수 해석 예시;
# coef_ = [0.8 (사용빈도), -0.5 (최근접속일), 0.3 (불만건수)]

# 해석:
# - 사용빈도가 1 표준편차 증가 → 이탈 log-odds 0.8 증가 → 이탈 확률 ↑
# - 최근접속일이 1 표준편차 증가 → 이탈 log-odds 0.5 감소 → 이탈 확률 ↓ (최근에 접속했으면 이탈 안 함)
# - 불만건수가 1 표준편차 증가 → 이탈 log-odds 0.3 증가 → 이탈 확률 ↑
```

> PM가치
> "어떤 특성이 얼마나 영향을 미치는가"를 수치적으로 설명 가능
> 이것이 의사결정자에게 보고할 때 가장 강력한 근거가 됨.
